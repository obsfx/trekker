trekker

github | npm | dashboard | claude code

issue tracking for ai coding agents

why i built this

I built this after using beads for a while. beads does the job, but its codebase has grown quickly without enough care for what is happening inside. a task tracker is a simple application. it should not need thousands of lines of code.

I noticed AI coding agents kept losing track of what they were doing. They'd start a task, hit a bug, go down a rabbit hole, and forget the original goal entirely. They needed a way to write down their own work and come back to it.

Trekker is a simple CLI that agents can use to manage their own tasks. Create tasks, update status, add comments. The agent controls the todo list, so it stays focused even across sessions. Local SQLite database. No server, no accounts, no sync. Just a .trekker folder in your project.

install

$ npm install -g @obsfx/trekker
$ trekker init
✓ Project initialized: my-project

features

task management - create, update, and track tasks with priorities (0-5) and descriptions. uses JIRA-style IDs (TREK-1, TREK-2) that agents can easily reference.

epics & dependencies - group tasks into epics. set dependencies between tasks to define execution order. prevents circular dependencies.

local sqlite - data stays in your project. no accounts, no configuration, no sync needed.

toon output - every command supports --toon for structured, machine-readable output.

usage

$ trekker task create -t "Build user authentication" -p 1
✓ Task created: TREK-1

$ trekker task update TREK-1 -s in_progress
✓ Task updated: TREK-1

$ trekker task update TREK-1 -s completed
✓ Task updated: TREK-1

$ trekker task list
TREK-1 | completed   | P1 | Build user authentication

dashboard

visual kanban board that reads from the same SQLite database. drag tasks between columns, see what's in progress, track your agent's work visually.

$ npm install -g @obsfx/trekker-dashboard
$ trekker-dashboard -p 3000
Dashboard running at http://localhost:3000

setup for ai agents

add to your agent's context (system prompt, CLAUDE.md, or similar):

  • use trekker to track your work. run trekker quickstart to learn the commands.
  • before starting: check trekker task list and mark your task in_progress
  • after finishing: mark the task completed and add a comment with trekker comment add

now the agent has persistent memory. it can start a session, check its task list, and continue exactly where it left off.
