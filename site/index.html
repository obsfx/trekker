trekker

Issue tracking for AI coding agents. Local SQLite database. No server, no accounts, no sync.

GitHub npm Dashboard Claude Code

Why I built this

I built this after using beads for a while. It does the job, but its codebase has grown quickly without enough care for what's happening inside. A task tracker is a simple application. It should not need thousands of lines of code.

I noticed AI coding agents kept losing track of what they were doing. They'd start a task, hit a bug, go down a rabbit hole, and forget the original goal entirely. They needed a way to write down their own work and come back to it.

Trekker is a simple CLI that agents can use to manage their own tasks. Create tasks, update status, add comments. The agent controls the todo list, so it stays focused even across sessions.

Get started in seconds

Install globally, then initialize in your project directory.

$ npm install -g @obsfx/trekker

or with bun

$ bun install -g @obsfx/trekker

then initialize

$ trekker init
✓ Project initialized: my-project

Everything an agent needs

Task Management

Create, update, and track tasks with priorities (0-5) and descriptions. Uses JIRA-style IDs (TREK-1, TREK-2) that agents can easily reference.

Epics & Dependencies

Group tasks into epics. Set dependencies between tasks to define execution order. Prevents circular dependencies automatically.

Local SQLite

Data stays in your project. No accounts, no configuration, no sync needed. Just a .trekker folder in your project root.

TOON Output

Every command supports --toon for structured, machine-readable output that AI agents can parse reliably.

Simple commands

The CLI is designed to be intuitive for both humans and AI agents.

$ trekker task create -t "Build user authentication" -p 1
✓ Task created: TREK-1

$ trekker task update TREK-1 -s in_progress
✓ Task updated: TREK-1

$ trekker task update TREK-1 -s completed
✓ Task updated: TREK-1

$ trekker task list
TREK-1 | completed   | P1 | Build user authentication

Visual kanban board

Install the separate dashboard package to get a visual kanban board that reads from the same SQLite database. See what's in progress and track your agent's work visually.

$ npm install -g @obsfx/trekker-dashboard
$ trekker-dashboard -p 3000
Dashboard running at http://localhost:3000

Give your agent persistent memory

Add these instructions to your agent's context (system prompt, CLAUDE.md, or similar):

  • Use trekker to track your work. Run trekker quickstart to learn the commands.
  • Before starting: check trekker task list and mark your task in_progress
  • After finishing: mark the task completed and add a comment with trekker comment add

Now the agent has persistent memory. It can start a session, check its task list, and continue exactly where it left off.
